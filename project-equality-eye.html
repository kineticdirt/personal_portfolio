EQUALITY EYE

AI-Driven Hate Speech Detection for Social Impact

1st Place - Social Impact RoseHack @ UCR 2024 94% Accuracy
View on GitHub

Project Overview

EQUALITY EYE is a powerful Chrome extension that leverages advanced machine learning to detect and classify misogynistic content on Twitter (X) in real-time. Built during RoseHack 2024, this project secured first place in the Social Impact category by addressing the critical issue of online harassment and creating a safer digital environment.

Inspiration

Inspired by RoseHack's mission of empowering women in STEM, EQUALITY EYE was created to tackle the pervasive challenges that women face in online spaces. RoseHack stands as a beacon of transformative empowerment, fearlessly addressing obstacles and sculpting a future where women not only thrive but redefine excellence in science and technology.

How It Works

The system employs a sophisticated three-tier architecture:

  1. Chrome Extension Frontend: Automatically extracts text content from tweets in your Twitter feed as you browse.
  2. FastAPI Backend Server: Processes incoming tweet data and coordinates with the ML model for real-time classification.
  3. DistilBERT Model: Fine-tuned transformer model that analyzes content and identifies misogynistic language with 94% accuracy.

Technical Implementation

Machine Learning Model

We fine-tuned an open-source DistilBERT transformer model from Hugging Face, training it on carefully curated datasets of social media posts from Reddit compiled by academic researchers. The model achieved an impressive 94% accuracy rate on our test dataset.

The training process involved sophisticated preprocessing and feature extraction techniques, enabling the model to understand nuanced language patterns and contextual cues that indicate misogynistic content.

Browser Extension

Built using vanilla JavaScript, the Chrome extension seamlessly integrates with Twitter's interface. Key features include:

  • Automatic text extraction from tweet elements using DOM manipulation
  • Real-time API communication with the backend server
  • Visual indicators for flagged content
  • Minimal performance overhead to ensure smooth browsing

Backend Architecture

The FastAPI backend provides a robust, high-performance API layer that:

  • Handles concurrent requests from multiple users
  • Manages model loading and inference optimization
  • Implements caching strategies for improved response times
  • Provides RESTful endpoints for extension communication

Challenges & Solutions

Model Loading Issues

Our biggest technical challenge involved PyTorch's inability to properly load our specially defined DistilBERT class. After extensive debugging and research, we resolved this by restructuring the model's class definition and adjusting the serialization process.

Backend Deployment

We spent over 6 hours troubleshooting backend server deployment issues, including:

  • CORS configuration for cross-origin requests from the extension
  • Model file size limitations (the .bin file was too large for GitHub)
  • Server hosting and endpoint accessibility
  • Proper error handling and response formatting

Data Scarcity

Finding high-quality, labeled training data for misogynistic content detection was challenging. We overcame this by combining multiple academic datasets and implementing data augmentation techniques to improve model generalization.
